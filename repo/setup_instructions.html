Gesture Project Web Server Guide

A complete beginner-friendly guide to running the Sign Language Translation application via the interactive Flask Web UI.

🧠 Web Architecture & Workflow

This project uses a browser-based frontend that streams frames to a Python Flask backend, allowing for real-time sign language prediction natively in the browser without installing desktop software like Tkinter.

1. Web Camera Capture

The browser uses navigator.mediaDevices to capture camera frames and renders them to an HTML5 canvas.

2. REST API Pipeline

The browser converts the current frame into a Base64 string and POSTs it to the Flask backend's /api/predict endpoint.

3. Landmark Tracking

The backend decodes the image, runs MediaPipe to extract 21 hand landmarks, and plots them onto a skeleton canvas.

4. Deep Learning Prediction

A custom Keras CNN classifies the skeleton image. The backend then returns the recognized symbol back to the browser in real-time!

🛠️ Troubleshooting Guide

Camera Permission Denied (Browser):
If the web page says "Requesting camera permission..." forever or fails, ensure you click "Allow" on the camera prompt in your browser. On Windows, verify that your browser has permission to use the camera in Windows Privacy Settings.
ImportError: No module named 'asl_predictor'
If the Flask server crashes on startup, it means Python cannot find the root directory. Ensure you set the PYTHONPATH environment variable as shown in the run instructions (e.g., $env:PYTHONPATH=".").
